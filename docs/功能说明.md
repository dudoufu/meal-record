# 干饭日记 — 功能说明文档

> 版本：v1.0 · 2026年6月

---

## 一、产品概述

### 1.1 产品定位

一款极简的饮食记录微信小程序，帮助用户快速记录每日饮食，标记好吃程度，回顾饮食历史。

### 1.2 核心主张

- **快速记录**：10 秒完成一条记录，零门槛
- **清爽无压**：不记热量、不搞复杂计算，只关注吃了什么、好不好吃
- **随时回顾**：按日期浏览饮食记录，照片墙一目了然

### 1.3 用户价值

| 用户需求 | 产品方案 |
|----------|----------|
| 记录今天吃了什么 | 输入食物名称 + 选餐次 + 选好吃程度 → 3 步保存 |
| 看以前吃了什么 | 日期条 + 日历选择器，按日浏览 |
| 回顾照片 | 照片墙按日期分组展示 |
| 知道什么好吃 | 好吃程度排行榜 + 周统计分布 |

## 二、整体架构

### 2.1 页面结构

```
底部 Tab 导航
├── 🏠 首页
├── 📊 记录（日期浏览）
└── 👤 我的

独立页面（非 Tab）
├── 📸 记录饮食（FAB 按钮触发）
│   ├── 拍照上传 → AI 识别 → 编辑 → 保存
│   ├── 相册上传 → AI 识别 → 编辑 → 保存
│   └── 手动录入 → 直接填表单 → 保存

弹窗
├── 📷 记录详情（查看/编辑好吃程度/删除）
├── ✍️ 手动记录（表单填写）
└── 📷 全部照片（照片墙）
```

### 2.2 页面关系

```
启动 → 首页
       │
       ├── 点击「记录饮食」FAB → 记录页 → 保存 → 回首页
       │
       ├── 底部 Tab「记录」→ 日期浏览页
       │
       ├── 底部 Tab「我的」→ 个人页
       │       └── 点击「全部照片」→ 照片墙弹窗
       │
       └── 点击某条记录 → 详情弹窗
               ├── 修改好吃程度 → 即时保存
               └── 删除 → 确认后删除
```

## 四、好吃程度体系

### 4.1 等级定义

| 等级 | 标识 | 表情 | 颜色 | 含义 |
|------|------|------|------|------|
| amazing | t1 | 😋 超赞 | 金色 #f59e0b | 非常好吃，强烈推荐 |
| good | t2 | 🤩 好吃 | 绿色 #10b981 | 味道不错，满意 |
| okay | t3 | 😐 一般 | 蓝色 #6366f1 | 还行，无功无过 |
| bad | t4 | 😣 踩雷 | 红色 #ef4444 | 不好吃，踩雷了 |

### 4.2 交互方式

- **记录时选择**：在识别结果或手动记录表单中点选
- **记录后修改**：在详情弹窗中点击任意等级，即时更新
- **默认值**：默认为"好吃 🤩"

### 4.3 统计维度

- **今日好吃榜**（首页）：展示当天各等级的数量
- **本周好吃分布**（记录页-统计区）：彩色比例条
- **本月最受欢迎**（记录页-统计区）：按好评率排名

## 五、AI 识别（可选功能）

### 5.1 设计原则

- AI 识别是辅助功能，**不是核心流程的依赖**
- 识别成功 → 自动填入食物名称
- 识别失败/超时 → 用户手动输入
- 离线状态下完全可用

### 5.2 识别流程

```
用户拍照/选图
    ↓
前端压缩图片（≤200KB）
    ↓
上传到云存储
    ↓
云函数调用 AI 识别
    ↓
┌─── 成功 ───→ 自动填入食物名称
└─── 失败 ───→ 提示"未识别到食物，请手动输入"
```

### 5.3 免费方案建议

| 方案 | 费用 | 说明 |
|------|------|------|
| 初期不接 AI | 0 元 | 纯手动输入，功能完整可用 |
| 腾讯云慧眼（云开发） | 前 1000 次免费 | 接入简单，精度一般 |
| 大模型视觉（Claude/GPT） | 按量计费 | 精度最高，适合后期 |
